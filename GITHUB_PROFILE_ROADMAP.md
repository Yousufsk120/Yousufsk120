## README Style Guide

### Tone

- Research-first
- Professional
- Human but not casual
- Clear for both academic and industry audiences
- Avoid overstating coding identity
- Emphasize device fabrication, materials, characterization, and research vision

### Avoid

- “Full-stack developer” unless the repository truly needs it
- Claiming software engineering expertise as the central identity
- Random unstructured repo names
- Empty repositories without README files
- Overly long personal story in technical repos

### Prefer

- “Electronic Materials & Device Researcher”
- “Device Fabrication”
- “Materials Synthesis”
- “Characterization”
- “Device Physics”
- “Neuromorphic Computing”
- “AI-assisted Research”
- “Open Science”
- “Sustainable Innovation”
